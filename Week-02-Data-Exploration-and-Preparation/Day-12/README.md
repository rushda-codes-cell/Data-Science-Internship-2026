# Day 12 – Data Drift Detection

## Objective

The objective of Day 12 was to monitor changes in incoming data using Evidently AI. Detecting data drift helps maintain model reliability by identifying differences between historical and current datasets.

---

## Tasks Performed

### 1. Data Drift Analysis

Compared historical and current datasets.

The following tasks were completed:

- Loaded reference and current datasets.
- Compared feature distributions.
- Generated drift reports.
- Identified drifted features.

### 2. Data Quality Monitoring

Validated the quality of incoming data.

The following tasks were completed:

- Checked missing values.
- Verified feature consistency.
- Monitored statistical changes.
- Evaluated overall data quality.

---

## Tools & Libraries Used

- Python
- Jupyter Notebook
- Pandas
- Evidently AI

---

## Outcome

By the end of Day 12, a data drift monitoring system was implemented to continuously evaluate dataset quality and detect changes that could affect model performance.
