# Day 7 – Week 1 Checkpoint and MLflow Experiment Tracking

## Objective

The objective of Day 7 was to review the work completed during Week 1, evaluate the progress made, document the results, and track machine learning experiments using MLflow. This checkpoint ensured that the dataset, forecasting models, and experiments were properly organized before moving on to advanced modeling in Week 2.

---

### 1. Week 1 Review

Reviewed all tasks completed throughout Week 1 to ensure that each stage of the data science workflow was successfully implemented.

The completed tasks included:

- Dataset Selection
- Exploratory Data Analysis (EDA)
- Data Cleaning
- Feature Engineering
- Customer Segmentation
- Time Series Data Preparation
- Demand Forecasting using Prophet
- LSTM Model Development

### 2. EDA Report Preparation

Prepared a summary report highlighting key insights obtained during exploratory data analysis.

The report included:

- Dataset overview
- Missing value analysis
- Distribution analysis
- Correlation analysis
- Customer behavior insights

### 3. Cleaned Dataset Review

Verified that the cleaned and feature-engineered dataset was complete, consistent, and ready for the next phase of the internship.

### 4. MLflow Experiment Tracking

Tracked forecasting model experiments using MLflow.

The following activities were completed:

- Logged model parameters
- Recorded evaluation metrics
- Saved forecasting models
- Compared experiment results
